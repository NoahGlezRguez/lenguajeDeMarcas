### 🟢 Bloque 1: De XSD a XML (Infraestructura de Servidores)

Este esquema define la configuración de una red de servidores. Contiene tipos derivados complejos, restricciones sobre extensiones previas y un uso estricto de espacios de nombres.

**El código XSD (`infraestructura.xsd`):**
```xml
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema"
           targetNamespace="http://red-empresarial.local/sys"
           xmlns:sys="http://red-empresarial.local/sys"
           elementFormDefault="qualified">

  <xs:complexType name="TextoConUnidad">
    <xs:simpleContent>
      <xs:extension base="xs:string">
        <xs:attribute name="unidad" type="xs:string" use="required"/>
      </xs:extension>
    </xs:simpleContent>
  </xs:complexType>

  <xs:complexType name="MemoriaRAM">
    <xs:simpleContent>
      <xs:restriction base="sys:TextoConUnidad">
        <xs:minInclusive value="8"/>
        <xs:maxInclusive value="1024"/>
        <xs:attribute name="unidad" type="xs:string" fixed="GB"/>
      </xs:restriction>
    </xs:simpleContent>
  </xs:complexType>

  <xs:element name="datacenter">
    <xs:complexType>
      <xs:sequence>
        <xs:element name="servidor" maxOccurs="unbounded">
          <xs:complexType>
            <xs:sequence>
              <xs:element name="hostname">
                <xs:simpleType>
                  <xs:restriction base="xs:string">
                    <xs:pattern value="srv-[a-z]{3}-\d{2}"/>
                  </xs:restriction>
                </xs:simpleType>
              </xs:element>
              <xs:choice>
                <xs:element name="ip_v4" type="xs:string"/>
                <xs:element name="ip_v6" type="xs:string"/>
              </xs:choice>
              <xs:element name="ram" type="sys:MemoriaRAM"/>
            </xs:sequence>
            <xs:attribute name="rol" use="required">
              <xs:simpleType>
                <xs:restriction base="xs:string">
                  <xs:enumeration value="BBDD"/>
                  <xs:enumeration value="Web"/>
                  <xs:enumeration value="Backend"/>
                </xs:restriction>
              </xs:simpleType>
            </xs:attribute>
          </xs:complexType>
        </xs:element>
      </xs:sequence>
    </xs:complexType>
  </xs:element>
</xs:schema>
```

**Tus tareas:**
1. **Crea el documento XML:** Escribe un XML válido que contenga el elemento raíz `<datacenter>`. Declara correctamente el espacio de nombres.
Incluye al menos tres servidores: uno de base de datos con IPv4,
uno web con IPv6 y otro de backend. Asegúrate de cumplir con la restricción de la RAM y el patrón del hostname.
2. **Consultas XPath:** Basándote en el XML que vas a crear, escribe las rutas XPath para:
   * Seleccionar el `hostname` de todos los servidores que tengan el rol de "Web".
   * Seleccionar los nodos `<ram>` cuyo valor interno sea estrictamente superior a 32 (independientemente del rol del servidor).

### 🔵 Bloque 2: De XML a XSD (Expediente Académico con Contenido Mixto)

Aquí tienes un fragmento de un expediente de notas que utiliza dos espacios de nombres y contenido mixto. Tu trabajo será diseñar el esquema que imponga las reglas estrictas de este documento.

**El archivo XML:**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<ac:expediente xmlns:ac="http://instituto.edu/academico" 
               xmlns:al="http://instituto.edu/alumnos" 
               curso="2023-2024">
               
  <al:estudiante matricula="M-99381">
    <al:nombre>Lucía</al:nombre>
    <al:apellidos>García Ruiz</al:apellidos>
  </al:estudiante>
  
  <ac:calificaciones>
    <ac:modulo id="DAM104">
      <ac:nombre>Lenguajes de Marcas</ac:nombre>
      <ac:nota convocatoria="Ordinaria">8.5</ac:nota> 
      <ac:observaciones>El proyecto final de <strong>XML y XSD</strong> fue sobresaliente.</ac:observaciones>
    </ac:modulo>
  </ac:calificaciones>
  
</ac:expediente>
```

**Condiciones obligatorias para tu esquema XSD:**
1. El atributo `matricula` del estudiante debe cumplir el patrón: la letra 'M', un guion y exactamente 5 dígitos.
2. El elemento `<ac:nota>` **debe** modelarse usando una restricción de `simpleContent` (como vimos al principio de nuestra conversación). El texto debe ser un decimal entre 0.0 y 10.0, y debe mantener el
atributo `convocatoria` (que solo puede ser "Ordinaria" o "Extraordinaria").
3. El elemento `<ac:observaciones>` debe permitir contenido mixto (texto mezclado con la etiqueta `<strong>`).

**Tus tareas:**
1. **Crea el código XSD:** Escribe las definiciones necesarias para validar este XML. Puedes asumir un único esquema con `targetNamespace="http://instituto.edu/academico"`, tratando el prefijo `al:` como parte
del mismo documento para simplificar, o usar `import` si prefieres el desafío completo.
2. **Consultas XPath:** Basándote en este XML, escribe las rutas XPath para:
   * Obtener el valor de la `<ac:nota>` del módulo cuyo `id` sea "DAM104".
   * Contar cuántos módulos (`<ac:modulo>`) tienen una nota superior o igual a 5.0 en la convocatoria "Ordinaria".
