JS Y FORMULARIOS

El Evento de Envío (submit)
	Es el momento más importante.
	- 	Interceptar el envío: escuchar el evento 'submit'.
	- 	event.preventDefault(): Se usa para evitar que el formulario se envíe automáticamente y la página se recargue,
		y asi validar los datos primero con JS.

Validación de Datos
	- 	HTML5: Puedes usar atributos como required, minlength, maxlength, pattern y type="email" directamente en el HTML.
	-	JavaScript: Te permite hacer validaciones más complejas (ej. comprobar si una contraseña es fuerte) antes de llamar
		manualmente a form.submit().

Radio Buttons y Checkboxes
	-	Radio Buttons: Se usan para elegir una sola opción de un grupo (deben compartir el mismo name). Para saber cuál
		está marcado, recorres el grupo y buscas el que tenga .checked === true.
	- 	Checkboxes: Para opciones múltiples.
	- 	Tip importante: Si un checkbox no está marcado, no se envía en los datos del formulario.
	- 	Para obtener los valores de varios marcados, usa document.querySelectorAll('input[name="lenguaje"]:checked').

Elemento Select (Desplegables)
	- 	Valor seleccionado: selectElement.value te da el valor de la opción elegida.
	- 	Selección múltiple: Si el HTML tiene el atributo multiple, puedes obtener un array de las opciones elegidas filtrando
		aquellas donde option.selected sea verdadero.

Eventos de interacción: input vs change
	- 	input: Se dispara en tiempo real mientras el usuario escribe o cambia algo (ideal para búsquedas dinámicas).
	- 	change: Solo se dispara cuando el usuario termina de cambiar el valor y quita el foco del campo (hace clic fuera).
